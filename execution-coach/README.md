# Execution Coach

A landing page (`index.html`) plus a working chat app (`app.html`) for the AI
Execution Coach. The app's replies are generated by Claude through a small
backend function in `api/chat.js` — your API key never touches the browser.

```
execution-coach/
├── index.html        landing page
├── app.html           chat app
├── assets/
│   ├── styles.css      shared design tokens + landing styles
│   ├── landing.js       hero countdown + diagnostic chip animation
│   ├── app.css          chat app styles
│   └── app.js            chat logic, sprint timer, localStorage
├── api/
│   └── chat.js         serverless function that calls the Anthropic API
├── package.json
└── .env.example
```

## Why a backend function?

The Anthropic API key has to stay secret. If the browser called
`api.anthropic.com` directly, anyone could open dev tools and steal your key.
`api/chat.js` runs on the server, holds the key as an environment variable,
and is the only thing that talks to Anthropic. The browser only ever talks to
your own domain.

## Deploy on Vercel (easiest)

1. Get an API key from the [Anthropic Console](https://console.anthropic.com/) (Settings → API Keys).
2. Push this folder to a GitHub repo, or run `vercel` from inside it with the
   [Vercel CLI](https://vercel.com/docs/cli) installed.
3. In the Vercel dashboard for the project: **Settings → Environment Variables**
   → add `ANTHROPIC_API_KEY` with your key → redeploy.
4. That's it — `index.html` and `app.html` are served as static files, and
   `api/chat.js` is automatically deployed as a serverless function at
   `/api/chat`.

No build step, no framework, no `vercel.json` required.

## Deploy on Netlify

Netlify Functions use a slightly different export shape. Move `api/chat.js`
into `netlify/functions/chat.js` and change the last line from:

```js
module.exports = async (req, res) => { ... }
```

to the Netlify handler signature:

```js
exports.handler = async (event) => {
  const { messages } = JSON.parse(event.body || '{}');
  // ...same logic as before, building `cleanMessages` and calling fetch...
  return {
    statusCode: 200,
    body: JSON.stringify({ reply })
  };
};
```

Then set `ANTHROPIC_API_KEY` under **Site settings → Environment variables**,
and update `assets/app.js` to fetch `/.netlify/functions/chat` instead of
`/api/chat`.

## Running locally

Static hosting alone won't run `api/chat.js`. The fastest way to test
locally is the Vercel CLI:

```bash
npm i -g vercel
vercel dev
```

This serves the whole folder, including the serverless function, at
`http://localhost:3000`.

## Customizing the coach

All of the coach's behavior — rules, tone, the seven-step framework, the
premium modes — lives in the `SYSTEM_PROMPT` constant at the top of
`api/chat.js`. Edit the text there; nothing else needs to change.

## Notes

- Conversation history and session stats are stored in the visitor's own
  browser (`localStorage`), not on a server. Clearing the conversation in the
  app, or clearing browser storage, deletes it.
- `api/chat.js` caps the conversation sent to Anthropic at the last 30
  messages per request to keep costs predictable. Raise or lower the
  `.slice(-30)` value as needed.
- The pricing tiers on the landing page are presentational only — there's no
  billing wired up. Hook up Stripe or your payment processor of choice if you
  want to actually charge for tiers.
